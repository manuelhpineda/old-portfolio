Design Process

01

Goal identification - This is the initial stage of the design process. This is where I interview the client and ask strategic questions to determine what goals the new website needs to fulfill. This is the most important part of any web development process. If these questions aren’t all clearly answered in the brief, the whole project can set off in the wrong direction.

02

Wireframe creation - with the goals well defined it is time for me to start designing a wireframe for the website. The next step is to find some design inspiration and build a mockup of the wireframe. Wireframes provide a framework for storing the site’s visual design and content elements, and can help identify potential challenges with the design.

03

Content creation - now with all the wire framing done it is time to start creating the content for all the individual pages keeping in mind search engine optimization (SEO).

04

Development - Development involves the bulk of the programming work, as well as loading content. Take a strategic approach, and avoid future hassles by constantly testing as you go.

05

Launch - The purpose of the launch phase is to prepare the website for public viewing. This requires final polishing of design elements, deep testing of interactivity and features and, most of all, a consideration of the user experience.
